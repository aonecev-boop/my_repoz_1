# 10. Продакшен (Релиз)

## Чек-лист перед релизом

### Код

- [ ] Все тесты проходят
- [ ] Покрытие кода ≥ 80%
- [ ] Нет критических багов

## Версионирование

Используется **Semantic Versioning** (SemVer):
```
MAJOR.MINOR.PATCH
  │      │     └── Багфиксы
  │      └──────── Новые фичи (обратно совместимые)
  └─────────────── Ломающие изменения
```

## Откат (Rollback)

### Процедура отката
1. Определить необходимость отката
2. Откатить деплой на предыдущую версию
3. Восстановить БД из бэкапа (при необходимости)
4. Проверить работоспособность
5. Задокументировать причину отката

```bash
# Пример отката
git revert <commit-hash>
# или
docker pull <image>:<previous-tag>
```
