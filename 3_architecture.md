## 1. Обзор архитектуры

Приложение построено по **клиент-серверной архитектуре** и состоит из трёх независимых слоёв:

```
┌─────────────────────┐   ┌──────────────────────┐
│   Чат-виджет        │   │   Админ-панель        │
│   (Frontend)        │   │   (Frontend)          │
└────────┬────────────┘   └────────┬──────────────┘
         │   HTTPS / WebSocket     │   HTTPS
         └───────────┬─────────────┘
                     ▼
         ┌───────────────────────┐
         │   Backend (Node.js)   │
         │   API + AI Engine     │
         └───────────┬───────────┘
                     │
         ┌───────────▼───────────┐
         │   Слой данных          │
         │   JSON-файлы + SQLite  │
         └───────────────────────┘
```

| Слой | Назначение |
|------|------------|
| **Frontend — Виджет** | Чат-интерфейс для клиента на сайте (текст, голос, файлы) |
| **Frontend — Админка** | Управление базами данных, просмотр диалогов и заявок |
| **Backend** | API-сервер, AI-ядро, бизнес-логика, управление данными |
| **Слой данных** | Хранение баз, диалогов, сессий, заявок |

## 3. Детальное описание компонентов

### 3.1. Frontend — Чат-виджет

Встраиваемый компонент, подключаемый на сайт через `<script>`-тег.

| Модуль | Назначение |
|--------|------------|
| **FloatingButton** | Круглая кнопка (правый нижний угол) для открытия/закрытия чата |
| **ChatWindow** | Основное окно диалога: шапка, лента сообщений, панель ввода |
| **MessageList** | Рендер сообщений с разделением «бот / клиент», поддержка эмодзи, ссылок |
| **InputPanel** | Текстовое поле, кнопка «Отправить», кнопка микрофона, кнопка прикрепления |
| **VoiceRecorder** | Запись голоса через Web Speech API → текст → отправка как текстовое сообщение |
| **FileUpload** | Загрузка изображений/файлов проектов, отправка на `/api/upload` |
| **NotificationSound** | Воспроизведение звука при новом сообщении (если чат свёрнут) |

**Взаимодействие с бэкендом:**
- Отправка сообщений: `POST /api/chat`
- Отправка голоса (если серверный STT): `POST /api/voice`
- Загрузка файлов: `POST /api/upload`
- Получение ответов: Server-Sent Events (SSE) или polling

### 3.2. Frontend — Административная панель

Отдельная страница (или SPA), доступная по маршруту `/admin`.

| Модуль | Назначение |
|--------|------------|
| **AuthPage** | Страница логина (логин/пароль) |
| **DatabaseEditor** | Единый интерфейс для редактирования 4 баз данных (таблицы с кнопками добавить/редактировать/удалить) |
| **DialogsViewer** | Список всех диалогов с фильтрацией, просмотр полного диалога и собранного JSON клиента |
| **MeasureRequests** | Список заявок на замер (город, улица, телефон, время, имя клиента) |

**Взаимодействие с бэкендом:**
- Авторизация: `POST /api/admin/login`
- CRUD баз данных: `GET/POST/PUT/DELETE /api/admin/db/:dbName`
- Диалоги: `GET /api/admin/dialogs`
- Заявки: `GET /api/admin/measures`

### 3.3. Backend — API Router

Express.js маршрутизатор с группами эндпоинтов:

| Группа | Маршруты | Описание |
|--------|----------|----------|
| **Chat API** | `POST /api/chat` | Приём текстового сообщения, возврат ответа бота |
| **Voice API** | `POST /api/voice` | Приём аудиофайла, STT (Whisper), передача в чат |
| **Upload API** | `POST /api/upload` | Приём файла проекта, сохранение, уведомление |
| **Admin API** | `/api/admin/*` | CRUD баз данных, просмотр диалогов, заявок |
| **Auth API** | `POST /api/admin/login` | Аутентификация администратора |

**Middleware:**
- `rateLimiter` — ограничение частоты запросов
- `authMiddleware` — проверка JWT-токена для `/api/admin/*`
- `fileValidator` — валидация загружаемых файлов (размер, тип)

### 3.5. Backend — Session Manager

Управление сессиями и сбор данных клиента.

| Функция | Описание |
|---------|----------|
| **createSession(sessionId)** | Создание новой сессии с пустым JSON клиента |
| **getSession(sessionId)** | Получение текущей сессии (история, JSON) |
| **addMessage(sessionId, role, content)** | Добавление сообщения в историю |
| **updateClientData(sessionId, field, value)** | Обновление поля во внутреннем JSON клиента |
| **closeSession(sessionId)** | Завершение сессии, перенос в архив |

**Идентификация сессий:**
- По `sessionId` из cookie или localStorage клиента.
- Тайм-аут неактивной сессии: 30 минут.

### 3.6. Backend — Database Manager

Загрузка и обновление JSON-баз в памяти.

```
┌──────────────┐    watch     ┌──────────────────────┐
│ JSON-файлы   │ ───────────► │ In-memory кэш        │
│ на диске     │ ◄─────────── │ (объекты JS)         │
│              │    save      │                      │
└──────────────┘              └──────────────────────┘
```

| Механизм | Описание |
|----------|----------|
| **Загрузка** | При старте сервера все 4 JSON-файла загружаются в оперативную память |
| **Обновление** | При изменении через админ-панель: обновляется файл на диске → кэш перечитывается |
| **Доступ** | PromptBuilder берёт данные из кэша (мгновенно, без чтения диска) |
| **Hot-reload** | Опционально: `fs.watch` для автоматического перечитывания при ручном изменении файлов |

### 3.7. Backend — Notification Service

| Канал | Когда срабатывает |
|-------|------------------|
| **Email** | Клиент прислал проект, запросил менеджера, записался на замер |
| **Telegram** | То же самое — дублирование в Telegram-бот менеджера |

Содержимое уведомления:
- Имя клиента
- Тип события (проект / менеджер / замер)
- Собранный JSON клиента
- Ссылка на полный диалог в админке

## 6. Формирование системного промта

Ключевой архитектурный элемент — **динамическая сборка промта** при каждом запросе к ИИ.

```
СИСТЕМНЫЙ ПРОМТ = АЛГОРИТМ + БАЗЫ ДАННЫХ
```

### 6.1. Структура промта

```

│                                                      │

│  │  - Сбор данных / JSON (часть 2)                │  │
```

### 6.2. Принцип обновления

| Что обновилось | Что происходит |
|----------------|----------------|
| Цена модели в админке | → JSON-файл перезаписан → кэш обновлён → следующий промт содержит новую цену |
| Информация о компании | → То же самое — мгновенное обновление |
| Алгоритм работы | → Редактирование файла промта (или пересохранение через админку) |

## 8. Безопасность

### 8.1. Схема аутентификации (админка)

```

  │  POST /api/admin/login             │

  │ ───────────────────────────────►   │
  │                                    │ bcrypt.compare(password, hash)
  │  ◄───────────────────────────────  │
  │  { token: JWT }                    │
  │                                    │
  │  GET /api/admin/db/models          │
  │  Authorization: Bearer {JWT}       │
  │ ───────────────────────────────►   │
  │                                    │ authMiddleware → verify JWT
  │  ◄───────────────────────────────  │
  │  { data }                          │
  ▼                                    ▼
```

### 8.2. Защита чат-API

| Мера | Описание |
|------|----------|
| Rate limiting | Не более 20 сообщений в минуту на IP |
| Валидация входа | Максимальная длина сообщения — 2000 символов |
| Санитизация | Очистка от XSS в сообщениях перед сохранением |
| CORS | Разрешены запросы только с домена `vstroy-mebel.ru` |
| Размер файлов | Загрузка файлов — максимум 10 МБ, только jpg/png/pdf |

## 9. Встраивание виджета на сайт

Виджет подключается одной строкой в HTML сайта:

```html
<!-- Чат-виджет АлИИна -->
<script src="https://chat.vstroy-mebel.ru/embed.js" defer></script>
```

Скрипт `embed.js`:
1. Создаёт `<iframe>` или вставляет DOM-элементы виджета.
2. Изолирует стили виджета от основного сайта (Shadow DOM или iframe).
3. Управляет состоянием: свёрнут/развёрнут, новое сообщение.

## 10. Масштабирование и развитие

### 10.1. Текущая архитектура (MVP)

- Один сервер Node.js
- JSON-файлы для баз данных
- SQLite для диалогов
- Монолитный backend

### 10.2. Возможности масштабирования

| Направление | Решение |
|-------------|---------|
| Рост нагрузки | Переход на PostgreSQL, Redis для сессий |
| Много серверов | Вынос сессий в Redis, балансировка (nginx) |
| Большие базы | Переход от JSON в промте к RAG (Retrieval-Augmented Generation) |
| Аналитика | Добавление дашборда: конверсия, популярные модели, средний чек |
| Мультиканальность | Подключение Telegram-бота, WhatsApp, VK |
